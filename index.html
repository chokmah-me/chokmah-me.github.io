Direct Response to Zach’s Compliance Objectives

1. What Zach Must Accomplish

  1. Collect every standard that DoE / CISA push to customers
  2. Collect each customer’s private “laundry list” of cyber clauses
  3. Cross-correlate the two piles so the SAME control satisfies the MAX number of sources (min-invest / max-coverage)
  4. Turn the resulting “golden” control set into formal product requirements that engineering can build and test against
  5. Keep the mapping alive as standards and customer contracts change

2. Where the Existing 4-Session Plan Already Covers This

  • ✔Session 2 AI-compliance tools (claude-compliance / kimi-compliance) automate multi-framework mapping, gap analysis, and evidence strategy.
  • ✔Output is already a YAML spec that engineering can treat as formal product requirements.
  • ✔Tools accept arbitrary framework names — feed them CISA, DoE, or customer-specific PDFs/spreadsheets.

3. Gaps & Recommended Extensions

  • ❖Harvesting standards: Course assumes documents are already on hand. Add a quick module on scraping DoE/CISA sites and extracting clauses from RFPs.
  • ❖Cost/effort optimisation: Mapping is covered, but not quantitative “biggest bang for buck” scoring. Extend with effort-weighting prompts.
  • ❖Customer prose normalisation: Tools work best on numbered controls. Add a pre-step to turn contractual prose into comparable controls.
  • ❖Change tracking: No built-in diff workflow yet. Add a “re-run on updated source” pattern and version-control tips.
